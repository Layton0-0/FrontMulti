Form

사용자로부터 어떠한 값을 받을 목적으로 기술
전달될 데이터는 form 태그 안에 포함되어야 한다.
get: 256Byte ~ 4000Byte 이내 크기의 데이터만 전송 가능.
주소줄에 address?key=value&key=value...
뉴스에 들어갈 때 뉴스 홈만 나오는게 아닌 특정 기사가 나오려면 뒤에 주소값이 붙어야 함.
그럴 때에 사용.
post: 크기 상관없음.
body 처음에 붙어서 넘어간다. (내용을 볼 수 없음.)
주소줄에 안 붙어서 넘어가지만 안의 데이터는 다 넘어감.

상품 주문

size는 필드의 크기(보이는 칸의 크기).
maxlength는 정의한 필드에 입력할 수 있는 최대값. 10이면 10자까지만 입력 가능.
required: 안 쓰면 전송 안됨. 무조건 전송.
readonly: 읽기 전용. 사용자가 값을 입력할 수 없음.
disabled: 지정한 action에 값을 전달하지 않음.
주문한 사람



이 히든 필드는 화면에 표시되지 않음.
사용자 모르게 서버에 어떠한 값을 전송할 목적으로 사용된다.
배송지








radio, checkbox를 하나로 묶은 형태(공간을 적게 차지함)
select value 속성이 없으면 option에 적은 문자열이 전송된다.
